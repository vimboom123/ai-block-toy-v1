# AI积木玩具状态机 Mermaid（拆成两张）v2

这版专门修一个问题：
**把“跑题拉回”收进帮助/异常块内部，别再被 Mermaid 甩到外面。**

---

## 图 1：主流程

```mermaid
---
config:
  theme: base
  layout: dagre
---
flowchart LR
    A([开始]) --> B[待机]
    B -->|唤醒词 / 按键| C[会话初始化]
    C --> D[角色开场]
    D --> E[任务发布]
    E --> F[等待孩子回应]
    F -->|收到孩子输入| G[输入理解]

    G -->|命中成功且需口头确认| H[口头确认是否完成]
    G -->|命中成功且无需确认| M[完成鼓励]

    H -->|确认完成| M
    H -->|还没完成 / 不确定 / 超时| I[进入帮助链]

    M --> N[判断下一步]
    N -->|还有下一任务| E
    N -->|没有下一任务| O[收尾总结]
    O --> P([正常结束])
```

## 图 2：帮助 / 兜底流程（修正版）

```mermaid
---
config:
  theme: base
  layout: dagre
---
flowchart TB

    A[等待孩子回应]
    C[输入理解]
    H[口头确认是否完成]

    subgraph HELP[帮助与拉回]
        direction TB
        C1[轻提示]
        D[明确提示]
        E[分步提示]
        F[示范]
        G[跑题拉回]
    end

    subgraph FALLBACK[兜底处理]
        direction TB
        B[重新唤回]
        I[家长接管暂停]
        J[安全停止]
        K[异常收尾]
        M([提前结束])
    end

    %% 等待 -> 唤回
    A -->|沉默超时| B
    B -->|重新响应| A
    B -->|再次沉默| D

    %% 输入理解 -> 帮助链
    C -->|第一次低置信 / 错答| C1
    C -->|第二次低置信 / 错答| D
    C -->|继续失败| E
    C -->|仍失败| F
    C -->|跑题| G

    %% 口头确认失败 -> 帮助链
    H -->|还没完成 / 不确定 / 超时| D

    %% 帮助链回流
    C1 -->|提示播报完成| A
    D -->|提示播报完成| A
    E -->|子步骤播报完成| A
    F -->|示范播报完成| A
    G -->|拉回后继续当前任务| A

    %% 家长接管
    F -->|示范后仍失败 / 需要成人帮助| I
    I -->|恢复| A
    I -->|结束| K

    %% 安全停止
    J --> K

    %% 异常结束
    K --> M
```

---

## 这版为什么会比上一版稳一点

- `跑题拉回` 不再孤零零挂在主图外侧
- 它现在被并进 `HELP` 子图里，和提示链一组
- 图方向改成 `TB`，能减少左右拉扯
- `重新唤回`、`家长接管`、`安全停止`、`异常收尾` 单独放进 `FALLBACK`
