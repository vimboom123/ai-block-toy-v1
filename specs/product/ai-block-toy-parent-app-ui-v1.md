# AI积木玩具家长端 App UI 主稿 v1

项目：AI积木玩具  
范围：D. 家长端手机 App UI / 信息架构 (IA)  
日期：2026-03-16  
状态：v1 正式归档版

## 1. 设计前提与设备规范

- **设备定位**：手机 App，竖屏（Portrait）优先，适配 iOS/Android 主流机型。
- **核心逻辑**：App 是后端投影层（Projection）的渲染器。它只消费脱敏、聚合、人话化后的 DTO，不运行业务逻辑。
- **视觉风格**：简洁、温馨、低认知负荷，强调“陪伴感”而非“控制台”。

## 2. MVP 优先级说明

在 v1 阶段，以下三个页面被定为 **MVP 第一优先级**：
1. **首页**：回答“现在能不能玩 / 刚才玩得怎么样”。
2. **会话页**：回答“现在孩子在干什么 / 我要不要帮一把”。
3. **报告页**：回答“这轮到底学到了什么 / 我接下来该跟孩子聊什么”。

这三页构成了“玩前决策 -> 玩中观察 -> 玩后互动”的价值闭环。内容页与设置页在 v1 采用极简实现。

## 3. 页面详细架构

### 3.1 首页 (Home)
- **页面目标**：提供当前状态的快速透视，引导进入核心交互路径。
- **首屏核心信息**：正在进行的会话状态（若有）或最新一份报告的成就摘要。
- **推荐模块顺序**：
    1. **实时状态卡片**：显示 `active_session` 的 `public_stage` 与完成进度条，提供“进入会话”入口。
    2. **最新报告预览**：显示 `latest_report` 的主题名、亮点标签（Achievement Tags）及简短总结。
    3. **内容继续入口**：显示上次未完成主题的断点续玩卡片。
    4. **异常状态提醒**：仅在设备离线或安全中止（Safety Stop）时显示的全局提醒条。
- **交互重点**：一键进入实时会话流；一键查看完整报告。
- **不该做成什么样**：不要做成复杂的设备管理后台；不要在首页罗列全量系统配置。

### 3.2 会话页 (Session)
- **页面目标**：实时同步玩具端的任务进展，作为家长的“数字观察窗口”。
- **首屏核心信息**：当前阶段文案（`public_stage`）、当前任务名称（`parent_label`）、任务进度数。
- **推荐模块顺序**：
    1. **阶段进度条**：可视化展示 `warming_up -> doing_task -> celebrating -> cooling_down`。
    2. **当前任务卡片**：展示 `parent_label`、`help_level_current`（如有提示则显示“系统引导中”）。
    3. **实时时间线 (Timeline)**：以气泡形式滚动显示 `session_timeline_view`。包含：系统指令摘要、孩子响应状态（非原始录音）、获得鼓励、提示触发。
    4. **家长介入提示 (Action Card)**：当 `help_level` 达到 `parent_takeover` 时置顶弹出，显示介入理由及建议动作。
    5. **控制操作区**：仅保留“暂停/恢复”及“安全结束”两个核心按钮。
- **交互重点**：实时性更新；介入提示的强弱交互（根据风险等级切换颜色）。
- **不该做成什么样**：不要显示原始 Transcript（隐私保护）；不要展示状态机节点名（如 `await_answer`）；不要让家长能远程触发玩具发声。

### 3.3 报告页 (Report)
- **页面目标**：将会话数据转化为家长的情感价值与互动建议。
- **首屏核心信息**：主题名快照、完成率（Completion Rate）、最高帮助等级、时长。
- **推荐模块顺序**：
    1. **报告总览**：图形化展示任务完成情况与整体表现评价。
    2. **亮点标签云**：提取 `achievement_tags`，如“坚持不懈”、“理解力强”。
    3. **任务拆解详情**：列表展示各 Task 的结果（`result_code`）与家长备注（`parent_note`）。
    4. **家长跟进建议**：提供 `follow_up_suggestion` 的一键复制或朗读参考，指导家长如何给孩子“回一口血”。
    5. **安全提醒区**：仅在有 `safety_notice_level` 时显示的合规说明。
- **交互重点**：分享报告卡片；跟进建议的话术预览。
- **不该做成什么样**：不要做成枯燥的 log 表格；不要展示 LLM 分类器的原始置信度数值。

### 3.4 内容页 (Content)
- **页面目标**：展示玩法资源池，降低家长的选课成本。
- **首屏核心信息**：推荐主题（基于年龄/进度）、内容分类导航。
- **推荐模块顺序**：
    1. **本周推荐/今日精选**：大图展示适合当前年龄段的主题。
    2. **主题卡片流**：显示 `name`、`age_band`、`difficulty_level`、`expected_duration`、`parent_goal_copy`（告诉家长这在练什么）。
    3. **已游玩记录**：按主题聚合的历史游玩频次与最高分。
- **交互重点**：主题详情弹窗中的“玩法预览”；“开始游玩”指令的确认。
- **不该做成什么样**：不要展示主题的版本号或内部 ID。

### 3.5 设置页 (Settings)
- **页面目标**：完成基础的设备绑定、儿童档案维护及通知偏好设置。
- **首屏核心信息**：设备连接状态、当前儿童 profile。
- **推荐模块顺序**：
    1. **设备状态卡片**：展示展示名、在线/离线、电量提醒。
    2. **儿童档案**：修改年龄段、兴趣标签。
    3. **通知设置**：报告推送、介入提醒、会话结束提醒的开关。
    4. **安全与内容边界**：设置内容敏感度等级。
- **交互重点**：简单的 Toggle 开关；极简的表单填写。
- **不该做成什么样**：不要做成全量的后台管理系统；不要开放复杂的 prompt 调试入口。

## 4. UI 状态口径对齐表

| 场景 | 对应 DTO 字段 | UI 表现建议 |
|---|---|---|
| **会话暂停中** | `session.status=paused` | 首页卡片显示“暂停中”，会话页按钮切换为“继续”。 |
| **系统给提示** | `public_stage=receiving_hint` | 会话页当前任务卡片叠加“引导中”动画效果。 |
| **任务成功** | `public_stage=celebrating` | 会话页触发短暂的全屏撒花或动效，随后切换任务。 |
| **安全终止** | `end_reason=safety_stop` | 首页弹出红色提醒条，点击查看处理建议。 |
| **完成确认** | `awaiting_child_confirmation=true` | 会话页可显示“等待孩子确认完成”，但不直接依赖内部状态名。 |

## 5. 交互约束限制

1. **不可撤销性**：一旦点击“安全结束”，会话立即结算并清理，不可在 App 端回撤。
2. **延迟容忍**：UI 需处理 1-2 秒的网络同步延迟，Timeline 采用流式加载或占位加载。
3. **隐私隔离**：App 绝不提供实时的音频监控流，仅提供文字摘要。
