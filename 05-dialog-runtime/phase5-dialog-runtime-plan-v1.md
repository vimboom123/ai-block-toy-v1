# Phase 5 - Dialog Runtime Integration v1

项目：AI积木玩具  
阶段：Phase 5 / 真实对话链接入  
日期：2026-03-17  
状态：启动中

## 1. 目标

这一阶段不再停留在 mock / fixture / contract 层，而是把“真实对话能力”接进主链。

核心目标：
1. 接入真实模型供应方（优先火山引擎 / 方舟）
2. 让主流程里的故事 / 引导 / 对话回复真正由模型生成
3. 保留最小可验证链路，避免接入后完全失控
4. 为后续语音播报 / 语音输入预留接口

## 2. 当前约束

- 第四步已收口，`software-e2e/` 为唯一 runnable path
- 页面尚未正式开工，因此本阶段先完成“对话运行时”与页面未来接口之间的最小闭环
- 本阶段先聚焦文本对话能力，不先深做语音

## 3. 本阶段主问题

要先搞清三件事：
1. 火山引擎 AccessKey / SecretKey 与方舟 API key 的职责边界
2. 真实对话调用链怎么嵌入现有主流程
3. 哪些场景信息必须在这一步接进来（如消防车场景）

## 4. 交付标准

至少应看到：
1. 第三方模型接入方式说明清楚
2. 本地安全配置位置明确
3. 一条最小真实对话调用链跑通
4. 页面开工时可直接接的 runtime 接口草案

## 5. 下一步

1. 识别火山引擎 AccessKey/SecretKey 与方舟 API key 对应的产品边界
2. 选择 phase 5 的主接入路径
3. 落一版 dialog runtime 接入说明
4. 再决定先接文本对话还是同步铺语音播报出口
